# Файл разработки

## Что сделано в этой итерации

1. Добавлен orchestration-слой `AppCoreModule` для «склейки» модулей в единые пользовательские сценарии:
   - `loginAndPrepare()` — логин, регистрация `device_id/device_user`, flush pending push-токенов, первичная метрика входа;
   - `connectVpn()` — метрики клика/успеха/ошибки подключения и возврат `session_id`;
   - `disconnectVpnAndFlushMetrics()` — корректный teardown VPN и отправка накопленных метрик в `/api/v1/metrics/client`;
   - `switchAccount()` — переключение аккаунта + отправка отложенных push token активного аккаунта;
   - `logoutActiveAccount()` — безопасный logout с disconnect и очисткой account-scoped данных через hooks.
2. Зарегистрированы cleanup hooks при logout (очистка `device_user` и push/inbox данных аккаунта).
3. Расширено покрытие unit-тестами:
   - happy-path end-to-end внутри core (login -> connect -> disconnect -> flush metrics);
   - проверка очистки данных после logout;
   - проверка корректной обработки pending push token после switch account.
